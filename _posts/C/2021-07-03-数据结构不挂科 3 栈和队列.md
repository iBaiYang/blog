### 模块1 栈

#### ⼩节1 栈的类型定义

**栈的定义**

也称为堆栈，是⼀种先进后出，删除和插⼊都在栈顶操作的线性表。

![]({{site.baseurl}}/images/20210708/20210708203202.png)

**栈的插入**

push()：在栈顶插⼊元素。

**栈的删除**

pop()：在栈顶移除⼀个元素，并将栈数-1。

`Push（&S，e）`     初始条件：栈S已存在   操作结果：插⼊元素e为新的栈顶元素

```
void push(SqStack s,ini e)
{
    if (s.top - s.base > s.stacksize) {
        s.base = (int*)realloc(s.base, (s.stacksize + incresize) * sizeof(int));
        s.top = s.base + s,stacksize;
        s.stacksize += incresize;

}
```

算法原理：
```
N = (N div d)×d + N mod d
```

⼗进制转⼋进制: `(1348)10 = (2504)8`

void conversion() {

        Push(S, N % 8);
        N = N / 8;

    }
}

**括号匹配的检验**

```
[ ( [ ] [ ] ) ]
1 2 3 4 5 6 7 8
```

检验括号是否匹配的⽅法可⽤ “期待的急迫程度”这个概念来描述。

出现不匹配的三种可能情况：

1.到来的右括弧不是所“期待”的；

2.到来的是“不速之客”；

3.直到结束，也没有到来所“期待”的括弧；

```
1.凡出现左括弧，则进栈；
2.凡出现右括弧，⾸先检查栈是否为空
若栈空，则表明该“右括弧”多余
否则和栈顶元素⽐较
若相匹配，则“左括弧出栈” 
否则表明不匹配；
3.表达式检验结束时，
若栈空，则表明表达式中匹配正确，
否则表明“左括弧”有余；
```

#### ⼩节3 栈类型的实现

**顺序栈**

类似于线性表的顺序映象实现，指向表尾的指针可以作为栈顶指针。

栈的顺序存储表示：
```
#define STACK_INIT_SIZE 100
typedef struct {
    SElemType *base;
    SElemType *top;
    int stacksize;
} SqStack;
```

构造⼀个最⼤空间为 maxsize 的空顺序栈 S：
```
Status InitStack (SqStack &S, int maxsize)
{
    s.base = new ElemType[maxsize];
    if (!S.base) {
        exit(OVERFLOW);  // 存储分配失败
    }
    S.top = S.base;
    S.stacksize = maxsize;
    return OK;
}
```

顺序栈的基本操作：

```
Status Push (SqStack &S, ElenType e)
{
    // 若栈不满，则将e插入栈顶
    if (S.top - S.base >= S.stacksize) {
        // 栈满
        return OVERFLOW;
    }
    *S.top++ = e;
    return ok;
}
```

```
Status Pop (SqStack &S, ElenType &e)
{
    // 若栈不空，则删除S的栈顶元素，
    // 用e返回其值，并返回ok，
    // 否则返回失败
    if (S.top == S.base) {
        return ERROR;
    }
    e = *--S.top;
    return ok;
}
```

### 模块2 队列

#### ⼩节1 队列的类型定义

**队列** 

和栈相反，是⼀种先进先出的线性表，它只允许在表的⼀端进⾏插⼊，⽽在另⼀端删除元素。

队尾：允许插⼊的⼀端

队头：允许删除的⼀端

**队列的类型定义**

```
wait add
```

**队列的八大基本操作**

1.InitQueue(&Q)

操作结果：构造⼀一个空队列列Q。

2.DestroyQueue(&Q)

初始条件：队列列Q已存在。

操作结果：队列列Q被销毁，不不再存在。

3.QueueEmpty(Q)

初始条件：队列列Q已存在。

操作结果：若Q为空队列列，则返回TRUE，否则返回FALSE。

4.QueueLength(Q)

初始条件：队列列Q已存在。

操作结果：返回Q的元素个数，即队列列的⻓长度。

5.GetHead(Q, &e)

初始条件：Q为⾮非空队列列。

操作结果：⽤用e返回Q的队头元素。

6.ClearQueue(&Q)

初始条件：队列列Q已存在。

操作结果：将Q清为空队列列。

7.EnQueue(&Q, e)

初始条件：队列列Q已存在。

操作结果：插⼊入元素e为Q的新的队尾元素。

8.DeQueue(&Q, &e)

初始条件：Q为⾮非空队列列。

操作结果：删除Q的队头元素，并⽤用e返回其值。

⽤用链表表示的队列列称为链队列列。⼀一个链队列列显然需要两个分别指示对头和对尾的指针才能唯⼀一确定。

**循环队列列**

在顺序队列列中，当队尾指针已经到数组的上界，不不能再有⼊入队操作，但其实数组中还有空位置，
这就叫做“假溢出”，解决假溢出的途径----采⽤用循环队列列。

循环队列的类型定义

```
wait add
```

**循环队列与循环链表的区别**

队列列（包括循环队列列）是一个逻辑概念，⽽而链表是一个存储概念。一个队列列是否是循环队列列，
不不取决于它将采⽤用何种存储结构，根据实际的需要，循环队列列可以采⽤用顺序存储结构，
也可以采⽤用链式存储结构，包括采⽤用循环链表作为存储结构。

本题考查堆栈和队列列的特点。堆栈和队列列是常⽤用的两种数据结构，其中队列列只允许在一端进⾏行行插⼊入，
另一端进⾏行行删除操作，具有先进先出的特征。⽽而堆栈只允许在同一端进⾏行行插⼊入和删除运算，具有后进先出的特征。
