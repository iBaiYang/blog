### 树的基本概念

#### 定义及特点

**树的定义**

树是N(N>=0)个结点的有限集合。

N=0时，称为空树

任意⼀棵⾮空树中应满⾜：

(1) 有且仅有⼀个特定的称为根的结点。

(2) 当N>1时，其余结点可分为m(m>0)个互不相交的有限集合T1，T2，...，Tm， 其中每个集合本身⼜是⼀棵树，称为根结点的⼦树

**树的特点**

树是⼀种递归的数据结构：

(1) 除根结点外的所有结点有且仅有⼀个前驱结点。

(2) 树中的所有结点可以有零个或多个后继结点。

#### 基本术语

![]({{site.baseurl}}/images/20210720/20210720203544.png)

**祖先结点**与**⼦孙结点**：根A到结点K的唯⼀路径上的任意结点，称为结点K的祖先结点。结点B是结点K的祖先结点，结点K是结点B的⼦孙结点。

**双亲结点**与**孩⼦结点**：路径上最接近结点K的结点E称为K的双亲结点，K为结点E的孩⼦结点。

**根**：A是树中唯⼀没有双亲的结点。

**兄弟结点**：有相同双亲的结点称为兄弟结点，如结点K和结点L。

**结点的度**：树中的⼀个结点的⾃结点个数称为该结点的度。

**树的度**：树中结点的最⼤度称为树的度。

**分⽀结点**：度⼤于0的结点称为分⽀结点。

**叶⼦结点**：度为0的结点称为叶⼦结点。

**结点的层次**：从根开始定义，根结点为第⼀层，它的⼦结点为第⼆层，依此类推。

**结点的深度**：从根结点开始⾃顶向下逐层累加。

**结点的⾼度**：从叶结点开始⾃底向上逐层累加。

**树的⾼度**：树中结点的最⼤层数。

**有序树**：树中结点的⼦树从左到右是有次序的，不能交换。

**⽆序树**：树中结点的⼦树从左到右没有次序，可以交换。

**路径**：树中两个结点之间的路径是由这两个结点之间所经过的结点序列构成的。

**路径⻓度**：路径上所经过的边的个数。

* 例题5-1  树最适合⽤来表示（ ）的数据。

A．有序   B．⽆序   C．任意元素之间具有多种联系。   D．元素之间具有分⽀层次关系


#### 性质

1. 树中的结点数等于所有结点的度数加1。
2. 度为m的树中第i层上⾄多有m^i-1个结点(i>=1)。
3. ⾼度为h的m叉树⾄多有(m^h-1)/(m-1)个结点。
4. ⾼具有n个结点的m叉树的最⼩⾼度为logm(n(m-1)+1) 向上取整。

A．n-1   B．n   C．n+1   D．2n

A．总和   B．最⼩值   C．最⼤值   D．平均值

### ⼆叉树

#### 基本概念

**二叉树的定义**

两个限制条件：
1. 每个结点⾄多有两棵树。
2. ⼦树有左右顺序之分，不能颠倒。

**特殊的二叉树**

满⼆叉树：棵⾼度为h，并且含有2^h-1个结点的⼆叉树。

![]({{site.baseurl}}/images/20210720/20210720204857.png)

完全⼆叉树：设⼀棵⾼度为h，有n个结点的⼆叉树，当且仅当其每⼀个结点都与⾼度为h的满⼆叉树中编号为其1～n的结点⼀⼀对应时，称为完全⼆叉树。

![]({{site.baseurl}}/images/20210720/20210720204945.png)

⼆叉排序树： ⼀棵⼆叉树或空⼆叉树，具有如下性质：
* 左⼦树上所有结点的关键字均⼩于根结点的关键字；
* 右⼦树的所有结点的关键字均⼤于根结点的关键字；

平衡⼆叉树：树上任⼀结点的左⼦树和右⼦树的深度之差不超过1。

**二叉树的性质**

1. ⾮空⼆叉树上叶⼦结点数等于度为2的结点数加1，即N0=N2+1。
   
2. ⾮空⼆叉树上第K层上⾄多有2^k-1个结点(k>=1)。
   
3. ⾼度为H的⼆叉树⾄多有2^H-1个结点(H>=1)。

4.对完全⼆叉树按从上到下、从左到右的顺序依次编号1，2，...，N，则：
* 当i>1时，结点i的双亲编号为i/2下取整
* 当2i<=N时，结点i左孩⼦编号为2i，否则⽆左孩⼦
* 当2i+1<=N时，结点i右孩⼦编号为2i+1，否则⽆右孩⼦
* 结点i所在层次（深度）为log2i下取整+1

5. 具有N个(N>0)结点的完全⼆叉树⾼度为log2(N+1)上取整或log2N下取整+1

* 例题5-4  下列关于⼆叉树的说法中，正确的是（ ）。

A．度为2的有序树就是⼆叉树。

B．含有n个结点的⼆叉树的⾼度为`[log2n]+1`。

C．在完全⼆叉树中，若⼀个结点没有左孩⼦，则它必是叶结点。

D．在任意⼀棵⾮空⼆叉树排序树中，删除某结点后⼜将其插⼊，则所得⼆叉排序树与删除前原⼆叉排序树相同。

* 例题5-5  设⾼度为h的⼆叉树上只有度为0和度为2的结点，则此类⼆叉树中所包含的结点数⾄少为（ ）。

A．h   B．2h-1   C．2h+1   D．h+1

**二叉树的存储结构「顺序存储」**

即⽤⼀个数组来存储⼀棵⼆叉树。

最适合于完全⼆叉树和满⼆叉树，⽤于存储⼀般⼆叉树会浪费⼤量的存储空间。

**二叉树的存储结构「链式存储」**

即⽤⼀个链表来存储⼀棵⼆叉树，⼆叉树中每⼀个结点⽤链表的⼀个链结点来存储。
包含数据域data，左指针域Ichild，右指针域rchild。

链式存储的结构体定义
```
typedef struct BiTnode
{
    ELemType data;
    struct BiTnode *lchild, *rchild;
} BiTnode, *BiTree
```

### 例题解析

* 解析5-1  D

树是⼀种分层结构，它特别适合组织那些具有分⽀层次关系的数据。

* 解析5-2  A

除根结点外，其他每个结点都是某个结点的孩⼦结点，因此树中所有结点的度数加1等于结点数，所以所有结点的度数之和就等于总结点数减1。

* 解析5-3  A

树的路径⻓度是所有路径⻓度的总和，树根到每个结点的路径的最⼤值应是树的⾼度-1。

* 解析5-4  C

A.⼆叉树是有序树，在⼆叉树中，若某个结点只有⼀个孩⼦结点，则这个孩⼦结点的左右次序是确定的；
⽽在度为2的有序树中，若某个结点只有⼀个孩⼦结点，则这个孩⼦结点就⽆须区分其左右次序。

B.其仅当为完全⼆叉树时才有意义，对于任意⼀棵⼆叉树，⾼度可能为`[log2n]+1~n`。

C.根据完全⼆叉树的定义，在完全⼆叉树中，若有度为1度结点，则只可能有⼀个，且该结点只有左孩⼦⽆右孩⼦。

D.在⼆叉排序树中插⼊结点时，⼀定插⼊在叶结点的位置，故若先删除分⽀结点再插⼊，则会导致⼆叉排序树的重构，其结果就不再相同。

* 解析5-5  B

结点最少的情况如图，除根结点层只有1个结点外，其他h-1层均有两个结点，结点总数=2（h-1）+1=2h-1。

![]({{site.baseurl}}/images/20210720/20210720205725.png)

* 解析5-6  B

⾮空指针数=总分⽀数=n-1，空指针数=2×结点总数-⾮空指针数=2n-（n-1）=n+1。
