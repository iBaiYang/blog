## WSL

上次安装Docker Desktop时，默认启用了WSL2。

如果没有启用WSL2，需要在命令行中执行以下命令。命令行以管理员身份打开。

1、启用WSL
```
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```

2、启用虚拟机功能
```
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

常用命令：
```
# 查看可安装发行版列表
wsl -l -o
# 更新WSL2内核（报错0x800701bc时执行）
wsl --update
# 查看WSL状态
wsl --status
# 设置WSL2为默认
wsl --set-default-version 2
# 启动Ubuntu
wsl -d Ubuntu-22.04
# 查看已安装子系统
wsl -l -v
# 关闭Ubuntu
wsl --terminate Ubuntu-22.04
# 卸载Ubuntu22.04
wsl --unregister Ubuntu-22.04
```

## 安装LLaMA-Factory

**命令列表**：
```
# 启动Ubuntu
wsl -d Ubuntu-22.04

# 切换到用户主目录，不然win10系统目录下，操作会报错。
cd ~

sudo apt-get update

sudo apt-get install -y python3-distutils

# 确定驱动是否已安装，否则搜索安装NVIDIA驱动程序
nvidia-smi

# 拉取代码
（这个地址不可用了，git clone https://gitee.com/hiyouga/LLaMA-Factory.git）
git clone https://gitee.com/ibaiyang/Llama-Factory.git

# 回退到指定版本
cd Llama-Factory
git reset --hard c244b1edb9af1697c3dfa5e7f3e352ca741ebcd6
cd ..

# 安装pip
sudo apt install python3-pip

# 安装uv虚拟环境管理包
sudo pip install uv -i https://mirrors.aliyun.com/pypi/simple

# 创建虚拟环境
uv venv finetune_01

# 激活虚拟环境
source finetune_01/bin/activate

# 到LLaMA-Factory目录
cd Llama-Factory

# 安装lamafactory。只想基础训练：.[torch]，想评估模型、绘图：.[metrics]，需要训练、评估指标、绘图：.[torch,metrics]，需要全量功能（包含训练、量化、推理、API、评估全部依赖）：.[torch,metrics,deepspeed,galore,awq,gptq,vllm,api]

uv pip install -e ".[torch,metrics]" -i https://mirrors.aliyun.com/pypi/simple

# 验证是否安装成功
llamafactory-cli version

# 验证GPU是否可用，应输出True
python -c "import torch;print(torch.cuda.is_available())"
```
