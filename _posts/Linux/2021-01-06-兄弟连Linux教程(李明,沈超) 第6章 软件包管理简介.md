### 正文

#### 一、软件包管理简介

1、软件包分类

源码包

    脚本安装包

二进制包（RPM包、系统默认包）

2、源码包

源码包的优点是：
* 开源，如果有足够的能力，可以修改源代码
* 可以自由选择所需的功能
* 软件是编译安装，所以更加适合自己的系统，更加稳定也效率更高
* 卸载方便

源码包的缺点
* 安装过程步骤较多，尤其安装较大的软件集合时（如LAMP环境搭建），容易出现拼写错误
* 编译过程时间较长，安装比二进制安装时间长
* 因为是编译安装，安装过程中一旦报错新手很难解决

3、RPM包

二进制包的优点
* 包管理系统简单，只通过几个命令就可以实现包的安装、升级、查询和卸载
* 安装速度比源码包安装快的多

二进制包缺点：
* 经过编译，不再可以看到源代码
* 功能选择不如源码包灵活
* 依赖性

#### 二、RPM包管理-rpm命令管理

**包命名与依赖性**

1、RPM包命名原则    

```
httpd-2.2.15-15.el6.centos.1.i686.rpm
httpd            软件包名
2.2.15           软件版本
15               软件发布的次数
el6.centos       适合的Linux平台
i686             适合的硬件平台
rpm              rpm包扩展名
```

2、RPM包依赖性

树形依赖：    a->b->c

环形依赖：    a->b->c->a

模块依赖：    
    
    模块依赖查询网站：                www.rpmfind.net

**安装升级与卸载**

1、包全名与包名

包全名：操作的包是没有安装的软件包时，使用包全名。而且要注意路径

包名：操作已经安装的软件包时，使用包名。是搜索/var/lib/rpm/中的数据库

2、RPM安装

rpm –ivh 包全名

选项：

```
-i（install）      安装
-v（verbose）      显示详细信息        
-h（hash）         显示进度
--nodeps          不检测依赖性
```
        
3、RPM包升级

rpm  -Uvh  包全名

选项：

```
-U（upgrade）      升级
```
    
4、卸载

rpm  -e  包名

选项：

```
-e（erase）      卸载
--nodeps        不检查依赖性
```

**RPM包查询**

1、查询是否安装

```
[root@localhost ~]# rpm  -q  包名
#查询包是否安装
选项：
        -q    查询（query）
[root@localhost ~]# rpm –qa
#查询所有已经安装的RPM包
选项：
        -a    所有（all）
```

2、查询软件包详细信息

```
[root@localhost ~]#  rpm –qi 包名
选项：
        -i    查询软件信息（information）
        -p    查询未安装包信息（package）   保全名
```

3、查询包中文件安装位置

```
[root@localhost ~]#  rpm –ql 包名
选项：
        -l    列表（list）
        -p    查询未安装包信息（package）   保全名
```

4、查询系统文件属于哪个RPM包

```
[root@localhost ~]#  rpm –qf 系统文件名
选项：
    -f    查询系统文件属于哪个软件包（file）
```

5、查询软件包的依赖性    

```
[root@localhost ~]# rpm –qR 包名
选项：
    -R    查询软件包的依赖性（requires）
    -p    查询未安装包信息（package）   保全名
```

**校验与文件提取**

1、RPM包校验

```
[root@localhost ~]# rpm –V 已安装的包名
选项：
    -V    校验指定RPM包中的文件（verify）
```

验证内容中的8个信息的具体内容如下：
* S    文件大小是否改变
* M    文件的类型或文件的权限（rwx）是否被改变
* 5    文件MD5校验和是否改变（可以看成文件内容是否改变）
* D    设备的主、从代码是否改变，major和minor号不一致
* L    文件路径是否改变
* U    文件的属主（所有者）是否改变
* G    文件的属组是否改变
* T    文件的修改时间是否改变

文件类型
* c    配置文件（config file）
* d    普通文档（documentation）
* g    “鬼”文件（ghost file），很少见，就是该文件不应该被这个RPM包包含
* l    授权文件（license file）
* r    描述文件（read me）

2、RPM包中文件提取

```
[root@localhost ~]# rpm2cpio 包全名  |  \
cpio -idv .文件绝对路径    

rpm2cpio              
#将rpm包转换为cpio格式的命令
cpio                    
#是一个标准工具，它用于创建软件档案文件和从档案文件中提取文件
```

```
[root@localhost ~]# cpio 选项 < [文件|设备]
选项：
        -i：copy-in模式，还原
        -d：还原时自动新建目录
        -v：显示还原过程
```

```
[root@localhost ~]# rpm -qf /bin/ls
#查询ls命令属于哪个软件包
[root@localhost ~]# mv /bin/ls /tmp/
#造成ls命令误删除假象
[root@localhost ~]# rpm2cpio /mnt/cdrom/Packages/coreutils-8.4-19.el6.i686.rpm | cpio -idv  ./bin/ls
#提取RPM包中ls命令到当前目录的/bin/ls下
[root@localhost ~]# cp /root/bin/ls  /bin/
#把ls命令复制会/bin/目录，修复文件丢失
```

#### 三、RPM包管理-yum在线管理

**6.3.1 IP地址配置和网络yum源**

rpm管理最大的问题就是依赖性，yum解决了上述的问题。

yum管理的同样是rpm包。

rpm管理和yum管理区别：

rom命令管理：手工管理

yum在线管理：自动管理

 1：配置IP
 
ifconfig  eth0 192.168.1.1

ifconfig只能配置ip和子网掩码，这个不是永久生效的，重启就会恢复原状。

想永久生效，可用setup命令，不过setup命令是redhat系列特有的。

2：网络yum源

CentOs网络yum源默认已经配置好，只要能上网，无需再更改。

能连网用CentOs-Base.repo

不能连网用CentOs-Media.repo

上述都是在 /etc/yum.repos.d目录之下的，大家可以进去看看还有哪些文件。

**6.3.2 yum命令**

1、常用yum命令

1）查询

查询所有可用软件包列表

> [root@localhost yum.repos.d]# yum list

搜索服务器上所有和关键字相关的包

> [root@localhost yum.repos.d]# yum search 关键字

2）安装

> [root@localhost yum.repos.d]# yum –y install 包名

选项：

    install    安装
    -y        自动回答yes

3）升级

> [root@localhost yum.repos.d]# yum -y update 包名

选项：

    update    升级
    -y        自动回答yes

4）卸载

> [root@localhost yum.repos.d]# yum -y remove 包名

选项：

    remove    卸载
    -y        自动回答yes

2、YUM软件组管理命令

列出所有可用的软件组列表

> [root@localhost ~]# yum grouplist

安装指定软件组，组名可以由grouplist查询出来

> [root@localhost ~]# yum groupinstall 软件组名

卸载指定软件组

> [root@localhost ~]# yum groupremove 软件组名

注意：若软件组名中有空格，那么名字要用” “引号，引起来！

**6.3.2 光盘yum源搭建**

光盘yum源的搭建步骤

1：挂载光盘–>用mount查看挂载信息

mount  /dev/cdrom   /mnt/cdrom/

2：让网络yum源失效

做法：通过mv命令将其他三个yum源后缀改掉

yum在线管理,光盘yum源的搭建,linux学习

3：修改光盘yum源文件

1）enabled = 1

2）baseurl = file:///mnt/cdrom（地址是你自己的关盘挂载地址）

3）# file:///media/cdrom/

`# file:///media/cdrecorder/`

注释这两个不存在的地址。

注意：配置文件要遵守Linux格式，否则会报错，重要！

#### 四、源码包管理

**6.4.1 源码包和RPM包的区别**

1：源码包和RPM包的区别--安装之前的区别：概念上的区别

安装之后的区别：安装位置的区别

2：源码包和RPM包的区别--rpm包安装位置

rpm包默认的安装路径

/etc/  配置文件安装路径

/usr/bin/可执行的命令安装目录

/usr/lib/程序所使用的函数库保存位置

/usr/share/doc/基本软件使用手册保存目录

/usr/share/man/帮助文件保存手册

3：源码包和RPM包的区别--源码包安装位置

安装指定位置当中，一般是/usr/local/软件名/…

**6.4.2 源码包的安装过程**

1：源码包的安装过程--安装准备

1）安装C语言编译器（gcc）

2）下载源码包

源码包安装和rpm包安装不会起冲突，因为两个安装位置不同

2：源码包的安装过程--安装注意事项

1）源码包保存位置（/usr/local/src/）

2）软件安装位置（/usr/local/）

3）如何确定安装过程报错（1：安装过程停止。2：出现error，warning或者no的提示）

安装过程是源代码编译过程

3：源码包的安装过程--源码包安装过程

1）下载源码包

2）加压缩下载的源码包

3）进入解压缩目录（注意：这一步不能丢）

进入之后

INSTALL是安装说明

README是使用说明

.configure软件配置与检查

1、定义需求的功能

2、检测系统环境是否符合安装要求

3、把定义好的功能选项和检测系统环境的信息都写入Makefile文件用于后续的编辑

.configure   –  –  help 查看帮助信息

.configure   –  –  prefix = /usr/local/apache2(这个是自己定义)（这个重要！！）

注意：

1：在同一台服务器上rpm包的apache和源码包的apache不可以同时开启，因为都占用80端口，同时开启两个会报错。

2：若想卸载，直接卸载这个目录即可，不会留任何垃圾。

如何选择源码包还是RPM包？

对外访问用源码包，本机使用用rpm包。

#### 五、脚本安装包与软件包选择

1：脚本安装包

脚本安装包并不是独立的安装类型，常见安装的是源码包。

脚本安装包的执行，是人为的将安装过程写成自动安装脚本，只要执行脚本，定义简单的参数，就可以完成安装。非常类似于windows下的软件安装。

2：脚本安装包 -- webmin的作用

webmin是一个基于web的Linux系统的管理界面，可通过图形化方式设置用户账号，Apache，DNS，文件共享等服务。

3：脚本安装包 -- webmin安装过程

1）下载软件http://sourceforge.net/projects/webadmin/files/webmin/

2）加压缩并进入解压缩目录

3）执行安装脚本

SSL是安全套接字功能
